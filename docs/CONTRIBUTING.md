# Contributing

Semut Core is an early-stage foundation for Semut applications.

Contributions should focus on reusable coordination patterns rather than one-off features.

## Good Contributions

* Shared schemas
* Clear interfaces
* Privacy-preserving workflows
* Reusable modules
* Tests
* Documentation
* Simple examples

## Avoid

* Domain-specific logic inside core
* Hidden automation
* Broad data access without need
* Complex abstractions before real use cases exist

## Design Rule

Build from real Semut applications.

Reusable code should emerge from actual needs in projects such as:

* semut-trace
* semut-tennis
* semut-business
