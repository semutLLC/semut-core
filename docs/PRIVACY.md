# Privacy Principles

Semut Core should minimize unnecessary access to user data.

## Principles

1. Collect only what is needed.
2. Prefer user-forwarded data over full account access.
3. Keep workspaces separated.
4. Make permissions explicit.
5. Allow users to delete traces.
6. Require approval for sensitive actions.
7. Avoid silent automation for payments or registrations.

## Email Privacy

Preferred methods:

1. Manual forwarding
2. Email rules created by the user
3. Dedicated Semut workspace email
4. OAuth only when necessary

Semut should not require full inbox access for basic functionality.

## Human Approval

Sensitive actions should require user approval.

Examples:

* spending money
* registering for a league
* sending external messages
* changing account settings
